# VCC Directory Content Publisher

Use this skill when Hermes needs to manage bilingual content posts for the VCC Directory website.

The current production site origin is:

```text
https://vccdir.com
```

## Purpose

Hermes can use this skill to:

- List content posts
- Read a single content post
- Create draft or published content
- Update content, including publishing and unpublishing
- Delete content
- Produce rich-text article bodies that match the website editor and public article renderer

## Authentication

All API requests require Bearer Token authentication.

Required Hermes environment variables:

- `VCC_BASE_URL`: site origin, usually `https://vccdir.com`
- `VCC_HERMES_API_TOKEN`: the same secret configured in the Worker as `HERMES_API_TOKEN`

Request headers:

```http
Authorization: Bearer ${VCC_HERMES_API_TOKEN}
Content-Type: application/json
```

If the token is missing or wrong, the API returns:

```json
{ "error": "Unauthorized" }
```

If the Worker still has the placeholder token, the API returns:

```json
{ "error": "HERMES_API_TOKEN is not configured" }
```

Never include the token in generated content, logs, public pages, or repository files.

## Content Model

The site stores bilingual content in `content_posts`.

Fields:

- `id`: number, generated by D1
- `title_zh`: required Chinese title
- `title_en`: required English title
- `slug`: URL slug, lowercase ASCII with hyphens is preferred
- `excerpt_zh`: optional Chinese excerpt
- `excerpt_en`: optional English excerpt
- `body_zh`: required Chinese rich-text HTML or plain text
- `body_en`: required English rich-text HTML or plain text
- `status`: `draft` or `published`
- `published_at`: ISO datetime or null
- `created_at`: generated by D1
- `updated_at`: generated by D1

## Rich Text Body Rules

The admin editor is a lightweight rich-text editor. Hermes should send `body_zh` and `body_en` as HTML when formatting is needed.

Allowed HTML tags:

```text
p, br, strong, b, em, i, u, h2, h3, ul, ol, li, blockquote, a, hr
```

Allowed link hrefs:

```text
https://...
http://...
mailto:...
/relative-path
#anchor
```

Do not send:

- `script`
- `style`
- inline event handlers such as `onclick`
- iframes
- images
- arbitrary classes or styles
- JavaScript URLs

The website sanitizes content again before public rendering, but Hermes should still generate clean HTML.

Plain text is also accepted. If plain text contains blank-line-separated paragraphs, the site converts it to paragraph HTML for display.

Recommended rich-text body example:

```html
<h2>费用结构</h2>
<p>虚拟卡通常包含开卡费、充值手续费和月费。</p>
<ul>
  <li><strong>开卡费：</strong>创建卡片时收取。</li>
  <li><strong>充值手续费：</strong>入金时按比例收取。</li>
</ul>
<blockquote>实际费用请以平台最新公告为准。</blockquote>
```

## Endpoints

### List Content

```http
GET /api/admin/content
GET /api/admin/content?status=published&limit=50
GET /api/admin/content?status=draft&limit=50
```

Query parameters:

- `status`: optional, only `draft` or `published` is applied
- `limit`: optional, max 100, default 50

Response:

```json
{
  "results": [
    {
      "id": 1,
      "title_zh": "虚拟卡费用指南",
      "title_en": "Virtual Card Fee Guide",
      "slug": "virtual-card-fee-guide",
      "excerpt_zh": "快速了解开卡费、月费和充值手续费。",
      "excerpt_en": "A quick guide to issuance fees, monthly fees, and top-up rates.",
      "body_zh": "<h2>费用结构</h2><p>...</p>",
      "body_en": "<h2>Fee Structure</h2><p>...</p>",
      "status": "published",
      "published_at": "2026-07-02T12:00:00.000Z",
      "created_at": "2026-07-02 12:00:00",
      "updated_at": "2026-07-02 12:00:00"
    }
  ]
}
```

### Get Content

```http
GET /api/admin/content/{id}
```

Returns one post object. Returns `404` if not found.

### Create Content

```http
POST /api/admin/content
```

Required fields:

- `title_zh`
- `title_en`
- `body_zh`
- `body_en`

Optional fields:

- `slug`: if omitted, the API generates one from `title_en` or `title_zh`
- `excerpt_zh`
- `excerpt_en`
- `status`: defaults to `draft`; only `published` publishes
- `published_at`: if omitted and status is `published`, the API sets the current time

Example:

```json
{
  "title_zh": "虚拟卡费用指南",
  "title_en": "Virtual Card Fee Guide",
  "slug": "virtual-card-fee-guide",
  "excerpt_zh": "快速了解开卡费、月费和充值手续费。",
  "excerpt_en": "A quick guide to issuance fees, monthly fees, and top-up rates.",
  "body_zh": "<h2>费用结构</h2><p>虚拟卡通常包含开卡费、充值手续费和月费。</p>",
  "body_en": "<h2>Fee Structure</h2><p>Virtual cards often include issuance fees, top-up fees, and monthly fees.</p>",
  "status": "published"
}
```

Returns the created post with status `201`.

### Update Content

```http
PUT /api/admin/content/{id}
```

Send any editable fields. Existing fields are preserved when omitted.

Publishing behavior:

- Set `status` to `published` to publish.
- If `published_at` is omitted, the API keeps the existing publish time or sets the current time.
- Set `status` to `draft` to unpublish; this clears `published_at`.

Example:

```json
{
  "status": "draft"
}
```

### Delete Content

```http
DELETE /api/admin/content/{id}
```

Response:

```json
{ "ok": true }
```

## Public Website Behavior

Published posts are visible at:

```text
/content
/content/{slug}
```

Draft posts are visible only to logged-in admins on `/content` and in the admin dashboard. Hermes API can still list drafts with:

```http
GET /api/admin/content?status=draft
```

The public website uses:

- absolute canonical URLs from `SITE_URL`
- `BlogPosting` JSON-LD for article pages
- `Blog` JSON-LD for `/content`
- `BreadcrumbList` JSON-LD
- sanitized rich-text rendering

## Recommended Hermes Workflow

1. Generate bilingual title, excerpt, and rich-text body.
2. Generate a lowercase ASCII slug with hyphens.
3. Create as `draft` when review is needed.
4. Create or update as `published` when ready.
5. Verify with `GET /api/admin/content/{id}`.
6. For published posts, verify public visibility at `/content/{slug}`.

## Curl Templates

Create:

```bash
curl -X POST "$VCC_BASE_URL/api/admin/content" \
  -H "Authorization: Bearer $VCC_HERMES_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d @post.json
```

Update:

```bash
curl -X PUT "$VCC_BASE_URL/api/admin/content/1" \
  -H "Authorization: Bearer $VCC_HERMES_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d @post-update.json
```

Delete:

```bash
curl -X DELETE "$VCC_BASE_URL/api/admin/content/1" \
  -H "Authorization: Bearer $VCC_HERMES_API_TOKEN"
```

## Quality Checklist

Before publishing, Hermes should ensure:

- Chinese and English content both exist.
- The slug is stable and URL-safe.
- The excerpt accurately summarizes the article.
- HTML uses only allowed rich-text tags.
- Links use safe URLs.
- The post status is intentional: `draft` or `published`.
- Public factual claims are phrased cautiously when they may change.
